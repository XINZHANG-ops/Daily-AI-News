# IBM

## Overview
IBM entered the AI development platform race with Bob, an AI-first development partner that automates the full software development lifecycle beyond code generation. Unlike consumer-facing coding agents (Claude Code, Cursor, Copilot), Bob targets the organizational SDLC — the handoffs, reviews, compliance checks, and deployment pipelines that consume most enterprise engineering time. IBM's strategic position is as a neutral multi-model orchestrator: routing tasks to Claude, Mistral, IBM Granite, and other models based on task fit. This neutrality appeals to enterprises wary of vendor lock-in from Microsoft (Copilot defaults to OpenAI) or Anthropic (Claude Code is Claude-only). IBM's 80,000+ internal users provide a credible reference base for enterprise sales.

## Key Relationships
- **Anthropic**: Claude is one of the models Bob routes tasks to; IBM benefits from Claude's coding strengths without being dependent on Anthropic
- **Microsoft**: Competing visions — Microsoft's Copilot defaults to OpenAI models within Office ecosystem; IBM Bob offers multi-model neutrality as a differentiator
- **Mistral**: Mistral models included in Bob's multi-model routing; European model presence helps with EU data residency requirements
